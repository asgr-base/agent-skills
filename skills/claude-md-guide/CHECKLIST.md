# CLAUDE.md チェックリスト

## 作成時チェックリスト

### 構造

- [ ] WHY / WHAT / HOW の構成になっているか
- [ ] 60行以下を目指しているか（最大100行未満）
- [ ] 詳細は別ファイルに分離しているか
- [ ] ファイル参照は標準Markdown記法 `[表示名](相対パス)` を使用しているか

### 内容

- [ ] 開発コマンド（build, test, lint）が記載されているか
- [ ] 重要なアーキテクチャパターンが記載されているか
- [ ] 命名規則が記載されているか
- [ ] 予期しない動作の警告が記載されているか

### 除外確認

- [ ] コードフォーマット規則を含んでいないか（リンターの仕事）
- [ ] 自明な指示を含んでいないか
- [ ] 変更頻度の高い情報を含んでいないか
- [ ] 一般的な知識（Git使い方など）を含んでいないか

### 強調表現

- [ ] 重要ルールには `IMPORTANT:` を使用しているか
- [ ] 絶対禁止には `NEVER` を使用しているか
- [ ] 必須事項には `YOU MUST` を使用しているか

## レビュー時チェックリスト

### 効果確認

- [ ] 指示が無視されていないか
- [ ] 曖昧な指示がないか
- [ ] 冗長な説明がないか

### 問題と対策

| チェック項目 | 問題の兆候 | 対策 |
|--------------|------------|------|
| 指示の遵守 | 同じミスが繰り返される | 強調表現を追加 |
| 行数 | 100行を超えている | 詳細を別ファイルに分離 |
| 更新頻度 | 頻繁に編集が必要 | 可変情報を別ファイルに |
| 明確さ | 解釈の余地がある | 具体的な例を追加 |

## 定期レビュー項目

### 月次レビュー

- [ ] 遵守されていない指示はないか
- [ ] 不要になった指示はないか
- [ ] 新しく追加すべき指示はないか
- [ ] 別ファイルに分離すべき内容はないか

### 振り返りポイント

1. **効果測定**: Claudeが指示に従っているか
2. **削除検討**: 3回以上無視された指示は表現を変更
3. **追加検討**: 繰り返し口頭で伝えている内容は追加
4. **分離検討**: 参照頻度の低い詳細は別ファイルへ

## 新規プロジェクト用クイックスタート

### Step 1: 最小構成で開始
```markdown
# Project Name

## コマンド
- dev: `npm run dev`
- test: `npm test`

## 重要事項
- [最も重要なルール1つ]
```

### Step 2: 作業中に気づいた点を追加
- `#` キーで即座に追記

### Step 3: 1週間後にレビュー
- 行数確認
- 遵守状況確認
- 不要項目削除

### Step 4: 必要に応じて分離
- 60行を超えたら分離を検討
- `.claude/rules/` にモジュール作成
